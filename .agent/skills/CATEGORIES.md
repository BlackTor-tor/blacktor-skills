# Suggested Categories

Use short, broad categories so the repository stays predictable.

## Suggested categories

- `backup` — backups, snapshots, sync, restore
- `coding` — code review, refactor, generation, debugging helpers
- `automation` — repeatable workflows and operational task runners
- `ops` — deployment, servers, infra, maintenance
- `docs` — documentation generation, cleanup, publishing
- `data` — extraction, transforms, analysis, import/export
- `communication` — messaging, response drafting, outbound coordination

## Rule of thumb

If a skill could reasonably fit multiple categories, choose the category based on the main operational outcome, not every secondary capability.
